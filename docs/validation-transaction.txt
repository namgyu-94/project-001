══════════════════════════════════════════════════════════════════════════════════════════
  PHASE 2  결과 처리
══════════════════════════════════════════════════════════════════════════════════════════

  전체 통과   →  Agent 상태: ACTIVE          →  MARKETPLACE 공개 활성화
  일부 실패   →  Agent 상태: VALIDATION_FAILED →  실패 단계 + 원인 반환, 재시도 가능

  [실패 단계별 조치]
  ┌──────┬──────────────────────────────────────────────────────────────────────────┐
  │ V1   │ Kong Route / Service 등록 상태 확인, Kong GW 로그 점검                   │
  │ V2   │ Kong Admin API 에서 리소스 직접 확인, Plugin enabled 여부 점검            │
  │ V3   │ Agent Pod IP·포트 확인, Kubernetes NetworkPolicy 점검                    │
  │ V4   │ Pod KONG_AUTH_TOKEN 환경변수 재확인 → Rolling Restart                   │
  │ V5   │ Pod 로그 확인 (kubectl logs), LLM 모델 로드 실패 여부 점검               │
  │ V6   │ Agent Card JSON 스펙 수정 후 재배포, A2A 버전 확인                       │
  └──────┴──────────────────────────────────────────────────────────────────────────┘


══════════════════════════════════════════════════════════════════════════════════════════
  OPTIONAL  추가 제안 검증 항목
══════════════════════════════════════════════════════════════════════════════════════════

  V7. 응답 지연 확인  (Latency SLA)
  │
  ├─ 측정 방법 : V1 요청 시 응답 시간 함께 기록
  ├─ 기준      : P95 응답시간 < 2,000ms
  └─ 처리      : 초과 시 실패 처리 없이 WARNING 상태로 표시 (운영 판단 영역)

  V8. TLS / SSL 인증서 확인
  │
  ├─ 대상   : HTTPS 외부 URL 사용 Agent
  ├─ 확인   : 인증서 유효기간 30일 이상 남아 있는지, 도메인 일치 여부
  └─ 처리   : 만료 임박 시 WARNING, 인증서 오류 시 실패

  V9. 동시 요청 처리  (Concurrency Smoke Test)
  │
  ├─ 방법   : 소량 동시 요청 (예: 5 req) 발송 후 전체 200 응답 확인
  ├─ 목적   : 단일 요청 통과 후 동시 처리에서 crash 하는 케이스 사전 탐지
  └─ 처리   : 1건 이상 실패 시 WARNING (필수 실패 처리 여부는 정책 결정)

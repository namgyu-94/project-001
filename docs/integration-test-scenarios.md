## S-01. 로그인 및 역할별 초기 화면 진입

### TC-01-01. User 계정 로그인 및 기본 랜딩 패널 확인

| 항목 | 내용 |
|------|------|
| **사전조건** | 시스템이 정상 기동된 상태. 로그인 오버레이(`#loginOverlay`)가 표시 중 |
| **테스트 데이터** | ID: `user01` / PW: `Test1234!` |

**테스트 절차**
1. 브라우저에서 AgentMarket URL 접속
2. 로그인 화면에서 ID 입력란에 `user01` 입력
3. PW 입력란에 `Test1234!` 입력
4. `로그인` 버튼 클릭

**예상 결과**
- 로그인 오버레이가 사라지고 메인 화면이 표시된다
- 사이드바 `MY` 섹션이 자동으로 펼쳐진다
- `user-status`(내 현황) 패널이 기본으로 표시된다
- 사이드바 하단 계정 영역에 `user01` 이름·이메일·`User` 역할 배지가 표시된다
- Topbar 우측 역할 칩(`.user-chip`)에 `User` 표시

## S-02. 마켓플레이스 탐색 및 Agent 구독

### TC-02-01. 키워드 검색으로 Agent 필터링

| 항목 | 내용 |
|------|------|
| **사전조건** | `user01`로 로그인, `marketplace` 패널 진입 상태. 마켓플레이스에 5개 이상의 Agent 카드가 등록되어 있음 |
| **테스트 데이터** | 검색어: `법무` |

**테스트 절차**
1. 검색 입력창에 `법무` 입력
2. `검색` 버튼 클릭

**예상 결과**
- 이름·설명·태그에 `법무`가 포함된 Agent 카드만 표시된다
- 해당되지 않는 카드(예: 문서 요약, 코드 리뷰)는 숨겨진다

### TC-04-05. JWT Token 표시/숨김 토글

| 항목 | 내용 |
|------|------|
| **사전조건** | `문서 업로드` 패널 진입 상태 |
| **테스트 데이터** | JWT: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.test.signature` |

**테스트 절차**
1. JWT Token 입력란에 테스트 데이터 입력
2. 입력란 옆 `SHOW` 버튼 클릭
3. 다시 `HIDE` 버튼 클릭

**예상 결과**
- 기본 상태: 입력값이 `●●●●●●`으로 마스킹된다 (type=password)
- `SHOW` 클릭 후: 입력값이 평문으로 표시되고 버튼이 `HIDE`로 변경된다
